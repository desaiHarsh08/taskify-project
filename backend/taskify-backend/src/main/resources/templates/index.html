Taskify Software: Backend Service Overview

Taskify Software is a robust task management application designed to streamline task organization and enhance productivity. The application offers a seamless user experience through its well-architected backend and interactive frontend.

Technology Stack

  • Backend:

    Spring Boot (Java version 17, Spring Boot 3.3.3)

  • Frontend:

    React, Bootstrap 5.3, Framer Motion

  • Hosting:

    AWS EC2 Ubuntu instance

Key Features

  1. Task Management: Users can create and manage tasks effectively, ensuring all aspects of task tracking and completion are covered.
  2. Notifications: Taskify includes robust notification features, allowing users to receive updates and reminders via WhatsApp and email.
  3. Flexibility: The application supports different types of tasks and functions, ensuring adaptability to various user requirements.

Software Terminologies

  • Task: A unit of work or activity to be accomplished within the system, representing a specific objective or goal.
  • Function: An individual activity or operation within a task, consisting of various actions or steps required to complete a task.
  • Field: Additional information or attributes associated with a function, providing context or details about the function such as its purpose, inputs, outputs, dependencies, etc.
  • Columns: Data fields or parameters within the function field that need to be completed or filled in, helping to organize and track the progress of functions within tasks.

Task Types

  • New Enquiry: This type of task involves handling new inquiries or requests from customers or users, including activities like receiving, processing, and responding to inquiries, and potentially creating new records or entries in the system.
  • Service: This type of task is related to providing repair or maintenance services for existing products or systems, involving activities like diagnosing issues, scheduling repairs, assigning technicians or resources, tracking repair progress, and updating records upon completion.
  • Combined Task: This is a special type of task that can be converted from either a New Enquiry or Repair Service task on demand during the task processing cycle if required. It combines functions from both New Enquiry and Repair Service task types.
